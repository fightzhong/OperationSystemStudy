- 引入
```
进程间的互斥与同步其实也算是一种进程间的通信, 如信号量机制, 但是它们被称为低级进程通信, 低级的原因
是因为:
  <1> 效率低, 生产者每次只能向缓冲池放入一个产品, 而消费者也只能取出一个产品
  <2> 通信对用户不透明, OS只为进程之间的通信提供了共享存储器(即缓冲池), 而关于进程之间通信所需之共
      享数据结构的设置、数据的传送、进程的互斥与同步, 都必须由程序员去实现
```

## 进程通信的类型
- 共享存储器系统
```
<1> 基于共享数据结构的通信方式, 如在生产者-消费者问题中的有界缓冲区, 该方式仅适用于传递相对少量的
    数据, 属于低级通信
<2> 基于共享存储区的通信方式, 为了传输大量数据, 在内存中划出了一块共享存储区域, 诸进程可通过对该共
    享区的读或写交换信息, 实现通信, 数据形式和位置甚至访问控制都是由进程负责, 属于高级通信, 需要通
    信的进程在通信前, 先向系统申请获得共享存储区中的一个分区, 并将其附加在自己的地址空间中, 便可对
    其中的数据进行操作, 使用完成后即可归还共享存储区
```

- 管道(pipe)通信系统
```
描述: "管道"其实就是一个共享文件, 用于连接一个读进程和一个写进程以实现它们之间的通信, 又名pipe文件,
      向管道(共享文件)提供输入的发送进程(即写进程)以字符流形式将大量数据送入管道, 而接收管道输出的
      接收进程(即读进程)则从管道中接收(读取)数据, 由于发送进程(写进程)和接收进程(读进程)是利用管道
      进行通信, 故又称为管道通信
管道机制:
  <1> 互斥, 即当一个进程正在对pipe进行读/写操作时, 其它进程必须等待
  <2> 同步, 当写(输入)进程把一定数量的数据写入pipe时, 便去睡眠等待, 直到读(输出)进程取走数据后再
      把它唤醒, 相应的, 当读进程读的是空的pipe时, 也应当睡眠等待, 直到写进程将数据写入pipe后唤醒
  <3> 确定对方是否存在, 只有确定了对方已经存在时才能进行通信          
```

- 消息传递系统
```
描述: 进程间以格式化的消息为单位, 将通信的数据封装到消息中, 并利用操作系统提供的一组通信命令(原语)
      在进程间进行进行消息传递, 完成进程间的数据交换, 如计算机网络中, 消息即报文, 将报文封装成特定
      的结构(增加头信息等)即可在互联网上进行消息的传递

通信方式:
  <1> 直接通信方式, 是指发送进程利用OS提供的发送原语, 直接把消息发送给目标进程
  <2> 间接通信方式, 是指发送和接收进程, 都通过共享中间实体(邮箱)的方式进行消息的发送和接收, 完成进
      程间的通信
```

- 客户机-服务器系统
```
描述: 客户机-服务器系统的通信机制, 在网络环境的各个应用领域已经成为主流的通信机制, 其主要的实现方法
     分为三类: 套接字、远程过程调用和远程方法调用
```
